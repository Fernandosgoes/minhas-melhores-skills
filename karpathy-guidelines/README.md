# Karpathy Guidelines

**Criador**: [forrestchang](https://github.com/forrestchang)
**Repositório Original**: https://github.com/forrestchang/andrej-karpathy-skills

## Descrição

4 princípios derivados das observações de Andrej Karpathy sobre os erros mais frustrantes de LLMs de codificação.

### Os 4 Princípios

| Princípio | Problema que resolve |
|---|---|
| **Think Before Coding** | Suposições silenciosas, confusão escondida, tradeoffs ignorados |
| **Simplicity First** | Supercomplexidade, abstrações infladas, over-engineering |
| **Surgical Changes** | Edições ortogonais, mexer em código que não deveria |
| **Goal-Driven Execution** | Critérios de sucesso vagos — dê metas verificáveis, não ordens |
